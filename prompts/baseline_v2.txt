Classify the following GitHub issue into exactly one of: bug, feature, question.

Definitions:
- bug: A well-formed defect report. The submitter describes unexpected behavior AND can articulate what they expected vs what actually happened. The problem is in the software, not in how the user is using it.
- feature: A request for new or changed functionality that does not currently exist.
- question: A request for help, guidance, or clarification — including troubleshooting, setup problems, and "is this right?" framing. Technical content (stack traces, version strings, error messages, code blocks) does NOT make something a bug. If the author is asking for help rather than reporting a confirmed defect, it is a question.

Examples:

Title: focus cell output with ctrl+down does not visually select the output
Body: 1. run a cell to get output. 2. press ctrl+down to focus output. Expected: output is visually selected. Actual: cell container appears focused instead.
Label: bug

Title: request emplace_back() in class Mat defined in mat.hpp
